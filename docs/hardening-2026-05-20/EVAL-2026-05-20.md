# Phase I Live-Eval — Audit-Trail v2 (2026-05-20)

Regulator-grade validation of Ed25519 signatures, STH endpoint, RFC 6962
inclusion proofs, external anchoring, and tamper detection.  
Prereqs: `docker`, `docker compose`, `python 3.12`, `cryptography` pip package.

---

## A — Clean-state stack bring-up

```bash
# Start from zero volumes. If running again: docker compose down -v first.
cd /path/to/nomos

# Generate a 32-byte Ed25519 seed and a 32-byte HMAC key.
export SIGNING_SEED=$(python -c "import secrets; print(secrets.token_hex(32))")
export HMAC_KEY="ci-only-hmac-key-for-live-eval-32bytes+"

# Inject into the API and worker containers (add to .env or pass inline).
NOMOS_AUDIT_SIGNING_KEY=$SIGNING_SEED \
NOMOS_HASHCHAIN_HMAC_KEY=$HMAC_KEY \
NOMOS_DB_PASSWORD=eval-db-pw \
docker compose up -d --build
```

Wait until all services are healthy:

```bash
docker compose ps --format "table {{.Name}}\t{{.Status}}"
```

Expected output: every service shows `healthy` (api, worker, postgres, valkey, vault,
vault-init completed). Retry after 60 s if vault-init is still running.

If a service stays `unhealthy`: `docker compose logs <service> --tail 50`.

> NOTE: `NOMOS_AUDIT_SIGNING_KEY` and `NOMOS_HASHCHAIN_HMAC_KEY` are not yet
> generated by Vault-init (`vault/init-entrypoint.sh` does not write them to
> `nomos/secrets/audit`). Until that gap is closed, they MUST be supplied via
> ENV or `.env`. Track as GAP-A1.

## D — Pull STH, verify signature OUTSIDE the container

Run this in a fresh Python process on the host (no DB, no container access).
Replace `$AGENT_ID` and `$SIGNING_SEED` with the values from steps B and A.

```bash
python - <<EOF
import hashlib, json, sys
from cryptography.hazmat.primitives.asymmetric.ed25519 import Ed25519PrivateKey

SIGNING_SEED = "$SIGNING_SEED"   # 64-hex-char seed from step A
AGENT_ID     = "$AGENT_ID"
API          = "http://localhost:8060"

import urllib.request, http.cookiejar
jar = http.cookiejar.MozillaCookieJar("/tmp/nomos-cookie.txt")
jar.load(ignore_discard=True, ignore_expires=True)
opener = urllib.request.build_opener(urllib.request.HTTPCookieProcessor(jar))

resp = opener.open(f"{API}/api/agents/{AGENT_ID}/audit/sth")
sth = json.loads(resp.read())
print("STH:", json.dumps(sth, indent=2))

# Reconstruct public key from seed (same derivation as hash_chain.py).
priv = Ed25519PrivateKey.from_private_bytes(bytes.fromhex(SIGNING_SEED))
pub  = priv.public_key()

# Recompute the signed body (must match merkle.signed_tree_head body format).
body = f"{sth['origin']}\n{sth['tree_size']}\n{sth['root_hash']}\n{sth['timestamp']}"
body_hash = hashlib.sha256(body.encode()).hexdigest()

try:
    pub.verify(bytes.fromhex(sth["signature"]), body_hash.encode())
    print("ok=True  — STH signature VALID")
except Exception as e:
    print(f"ok=False — STH signature INVALID: {e}")
    sys.exit(1)
EOF
```

Expected output: `ok=True  — STH signature VALID`.

If `ok=False`: confirm the same `SIGNING_SEED` was used in step A (the seed
injected into the container must match what the verifier uses here).

## E — Inclusion proof for entry 0, recompute root

```bash
python - <<EOF
import hashlib, json, sys
import urllib.request, http.cookiejar

AGENT_ID = "$AGENT_ID"
API      = "http://localhost:8060"

jar = http.cookiejar.MozillaCookieJar("/tmp/nomos-cookie.txt")
jar.load(ignore_discard=True, ignore_expires=True)
opener = urllib.request.build_opener(urllib.request.HTTPCookieProcessor(jar))

proof = json.loads(opener.open(f"{API}/api/agents/{AGENT_ID}/audit/proof/0").read())
print("Proof:", json.dumps(proof, indent=2))

# Re-implement RFC 6962 verify_inclusion_proof (pure function, no DB).
LEAF_PFX = b"\x00"
NODE_PFX = b"\x01"

def hash_leaf(data: bytes) -> bytes:
    return hashlib.sha256(LEAF_PFX + data).digest()

def hash_node(l: bytes, r: bytes) -> bytes:
    return hashlib.sha256(NODE_PFX + l + r).digest()

# leaf_data = the chain entry's hash hex (as UTF-8 bytes), same as merkle.py.
# We need the entry's hash from the chain. Fetch it from the audit endpoint.
audit = json.loads(opener.open(f"{API}/api/agents/{AGENT_ID}/audit").read())
entry_hash_hex = audit["entries"][0]["chain_hash"]
leaf_data = entry_hash_hex.encode("utf-8")

node = hash_leaf(leaf_data)
fn, sn = proof["leaf_index"], proof["tree_size"] - 1
for sibling_hex in proof["audit_path"]:
    sibling = bytes.fromhex(sibling_hex)
    if fn % 2 == 1 or fn == sn:
        node = hash_node(sibling, node)
        while fn % 2 == 0 and sn > 0:
            fn >>= 1; sn >>= 1
    else:
        node = hash_node(node, sibling)
    fn >>= 1; sn >>= 1

computed_root = node.hex()
expected_root = proof["root_hash"]
match = computed_root == expected_root
print(f"computed root : {computed_root}")
print(f"proof root    : {expected_root}")
print(f"ok={match}  — inclusion proof {'VALID' if match else 'INVALID'}")
if not match:
    sys.exit(1)
EOF
```

Expected output: `ok=True  — inclusion proof VALID`.

If the proof returns 404: the agent has no entries yet. Re-run step C (anchor
writes a forward marker entry into the chain), then re-try.

## F — Tamper test: byte-flip in chain.jsonl, verify detects it

```bash
# 1. Find the chain file inside the named volume.
CHAIN_PATH=$(docker exec nomos-nomos-api-1 \
  python -c "
from nomos_api.config import settings
print(settings.agents_dir)
")
# CHAIN_PATH is /data/agents. Agent dir = /data/agents/<AGENT_ID>/audit/chain.jsonl

# 2. Copy the chain out, corrupt one byte in the first entry's data field.
docker cp nomos-nomos-api-1:/data/agents/$AGENT_ID/audit/chain.jsonl /tmp/chain-orig.jsonl
python - <<'PYEOF'
import json, pathlib
lines = pathlib.Path("/tmp/chain-orig.jsonl").read_text().splitlines()
first = json.loads(lines[0])
# Flip the first character of the data dict's string representation.
raw_data = first.get("data", {})
# Inject a bogus key — simplest byte-level corruption via JSON.
raw_data["__tampered__"] = True
first["data"] = raw_data
lines[0] = json.dumps(first, sort_keys=True, separators=(",",":"))
pathlib.Path("/tmp/chain-tampered.jsonl").write_text("\n".join(lines) + "\n")
print("tampered chain written to /tmp/chain-tampered.jsonl")
PYEOF

# 3. Overwrite the chain inside the container.
docker cp /tmp/chain-tampered.jsonl \
  nomos-nomos-api-1:/data/agents/$AGENT_ID/audit/chain.jsonl

# 4. Run nomos audit --verify inside the container.
docker exec \
  -e NOMOS_HASHCHAIN_HMAC_KEY="$HMAC_KEY" \
  -e NOMOS_AUDIT_SIGNING_KEY="$SIGNING_SEED" \
  nomos-nomos-api-1 \
  python -m nomos audit \
    --agent-dir /data/agents/$AGENT_ID \
    --verify
echo "Exit code: $?"
```

Expected output: `Audit chain INVALID` and one or more error lines mentioning
`hash mismatch`, `HMAC mismatch`, or `Ed25519 signature mismatch`. Exit code `1`.

If the chain reports `VALID` after tampering: the verify loop is not recomputing
the canonical hash from fields — file a P0 bug against `verify_chain` in
`hash_chain.py`.

Restore the original chain before continuing:

```bash
docker cp /tmp/chain-orig.jsonl \
  nomos-nomos-api-1:/data/agents/$AGENT_ID/audit/chain.jsonl
```

## G — Optional perf-smoke: 1000 entries, time anchor + Merkle

```bash
docker exec \
  -e NOMOS_HASHCHAIN_HMAC_KEY="$HMAC_KEY" \
  -e NOMOS_AUDIT_SIGNING_KEY="$SIGNING_SEED" \
  nomos-nomos-api-1 python - <<EOF
import time, asyncio
from pathlib import Path
from nomos_api.config import settings
from nomos.core.hash_chain import HashChain
from nomos_api.worker.jobs.audit_anchor import anchor_audit_heads

audit_dir = settings.agents_dir / "$AGENT_ID" / "audit"
chain = HashChain(storage_dir=audit_dir)

t0 = time.perf_counter()
for i in range(1000):
    chain.append("agent.action", "$AGENT_ID", {"i": i})
write_ms = (time.perf_counter() - t0) * 1000
print(f"1000 appends: {write_ms:.0f} ms  ({write_ms/1000:.2f} ms/entry)")

t1 = time.perf_counter()
asyncio.run(anchor_audit_heads(ctx=None))
anchor_ms = (time.perf_counter() - t1) * 1000
print(f"anchor + Merkle root: {anchor_ms:.0f} ms")
EOF
```

Acceptable thresholds (single-host Docker, 2024-era hardware):

| Metric | Acceptable |
|---|---|
| Per-entry append (Ed25519 sign + HMAC + SHA-256 + disk) | < 5 ms |
| 1000-entry anchor + MTH computation | < 2000 ms |

If per-entry time exceeds 5 ms: profile `_compute_signature` — Ed25519 sign
should be ~50 µs; disk I/O is the dominant factor on spinning disk.
