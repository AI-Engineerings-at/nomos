## 🟠 H-Phase — Architektur-Refactors (v0.5.0, ~8-12h)

### O3: services/chat_service.py extraction (C-F7)

`proxy.py` mischt 4 Verantwortlichkeiten:
1. HTTP-Route + RBAC
2. ContextPipeline orchestration
3. Memory.store_message persistence
4. Provider-Dispatch (Direct LLM vs Gateway)

Trennung:
- `services/chat_service.py::ChatService.execute(user, agent, message) → ChatResult`
- Router bleibt thin: RBAC + map result → HTTPException
- Tests können `ChatService` ohne httpx-Mocks fahren

**Aufwand:** 3-5h. Behavior-Tests müssen weiter grün bleiben.

### O4: Manifest Migration Framework (C-F11)

`manifest.py` hat `manifest_version: "1.0.0"` Feld aber **keinen
Migration-Pfad**. Kunden mit altem Manifest-Schema auf neuem NomOS-Tag
brechen hart.

```python
# nomos-cli/nomos/core/manifest_migrations.py
MIGRATIONS: dict[str, Callable[[dict], dict]] = {
    "1.0.0": _migrate_1_0_0_to_1_1_0,
    # ...
}

def migrate_manifest(data: dict) -> dict:
    version = data.get("manifest_version", "1.0.0")
    while version in MIGRATIONS:
        data = MIGRATIONS[version](data)
        version = data["manifest_version"]
    return data
```

Plus: `extra="ignore"` auf disk-persisted Schemas (C-F12 Forward-Compat-Falle).

**Aufwand:** 3-4h.

### O5: Plugin Contract-Check (C-F10)

`scripts/check-contracts.ts` validiert nur `nomos-console/src/lib/types.ts`.
Plugin in `nomos-plugin/src/api-client.ts` hat eigene Type-Definitionen
(`AuditEntry`, `ComplianceResult`, `BudgetResult`) die nicht aus
`schemas.py` generiert sind. Ein Backend-Refactor kann das Plugin
silent brechen.

Extension: zweites SourceFile parsen, eigene naming-map.

**Aufwand:** 2-3h.

## 🟢 V-Phase — Security Frontier (v0.6.0+)

### V1: Vault TLS-Listener (audit A-#16)

Heute Vault läuft mit `tls_disable:1`. Default. Breaking-Change für
dev-flows. Strategie:
- Listener auf 2 Ports binden (HTTP intern, HTTPS extern)
- Cert-Path konfigurierbar
- dev_mode lässt http durch, prod forces tls

### V2: Vault N-of-M Shamir Unseal (audit A-#16)

`-key-shares=1 -key-threshold=1` ist Default. Operator-grade braucht
3-of-5 oder KMS-auto-unseal-doc.

### V3: Gateway-Token Vault AppRole (audit A-#13)

Gateway liest Token aus `/vault/init/gateway-token` File-Mount —
TOCTOU + chmod-pflicht. Sauberer: Gateway nutzt eigenes Vault
AppRole.

### V4: Phase-B2 Sigstore Rekor Public Anchoring (opt-in)

`manifest.governance.publish_to_rekor: bool` (default false).
Wenn True: anchor cron schreibt auch in Sigstore Rekor public
transparency log. Data-Protection-Trade-off (chain-roots externalisiert),
deshalb opt-in.
